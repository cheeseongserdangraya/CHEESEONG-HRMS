CSV 备份还原工具

只有在 Supabase 资料真的不见了(项目被误删、表被误删等灾难情况)才需要用这个。把你之前从 Table Editor 导出的 CSV,原封不动地导回资料库。只有 admin 能用。

⚠️ 使用前必读

  • 这个工具是把 CSV 原封不动插回去,**保留原本的 id**,所以只适合「资料表是空的」这种情况(比如整个专案重建)
  • 如果表里已经有资料,重复导入同一份 CSV 会因为 id 重复而报错(这是保护机制,不会让你不小心导入两份)
  • **顺序很重要**:一定要先还原「员工资料」,再还原其他表——因为薪水、请假、报销记录都是靠 employee_id 关联到员工,员工资料没建好,其他表会插入失败
  • 每张表选对应的 CSV 文件,文件名不重要,重要的是内容欗位要跟原本这张表一致(直接用 Table Editor 导出的原始 CSV 就没问题)
